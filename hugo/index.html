Hugo

Hugo é um gerador de sites estáticos de código aberto escrito em Go. Ele permite criar sites estáticos de forma rápida e fácil, sendo uma ferramenta popular entre desenvolvedores web e criadores de conteúdo.

Comandos

new

1

hugo new site <site-name>

Ao executar esse comando, o hugo criará uma pasta e irá gerar um novo site nela. É útil para inicializar um novo projeto.

1

hugo new posts/<file-name>.md

Esse comando irá criar um novo arquivo. Neste caso, criará um novo post na pasta destino, contendo apenas informações de header. Por padrão ele cria um arquivo com propriedade draft: false, o que significa que o post é um rascunho e não será exibido na página.

Testando

Para testar seu site/blog, basta entrar no diretório contendo o arquivo hugo.toml e executar o comando

1

hugo serve

Esse comando vai criar um servidor local em uma porta aleatória em localhost para você visualizar seu site final antes de precisar compilar.

Compilando

Para compilar é muito simples. Basta digitar o comando na pasta raiz do projeto.

1

hugo

O hugo criará todos os arquivos web (.html, .css) prontos para serem hospedados.

Utilizar a flag -t <theme-name> em seguida faz o site ser compilado utilizando um tema em específico da pasta themes.

Pro-tips

Algumas dicas que eu aprendi na minha experiência utilizando a ferramenta.

Assets públicos em repositório submódulo

Por padrão, hugo criará os arquivos compilados na pasta public/ da raiz do projeto. Você pode manter seu código-fonte, com todos os posts e drafts em formato .md em um repositório privado e adicionar a pasta public/ como um submódulo para outro repositório público, onde sua página estará hospedada. Fazendo assim você pode ter liberdade de adicionar qualquer informação útil no arquivo md e subir sempre que quiser, e também subir para seu site público com facilidade.

Primeiro iremos criar uma pasta de submódulo indicando o url do repositório onde estará hospedada o conteúdo de public/ e em seguida a que será criada, ou seja, public também.

1

git submodule add -b main <url-do-repositório> public
